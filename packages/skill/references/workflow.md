# 询报价业务流程

## 角色

- **买方 (Buyer)**: 发起询价的一方（如修理厂、采购部门）
- **卖方 (Supplier)**: 接收询价并报价的一方（如供应商、经销商）

## 完整流程

```
┌─────────────────────────────────────────────────────────────────┐
│                         买方                                      │
├─────────────────────────────────────────────────────────────────┤
│  1. 创建询价单                                                    │
│     - 填写产品/零件名称                                           │
│     - 填写规格信息（OE号、车型、数量等）                           │
│     - 状态变为 pending                                            │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 询价推送
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                         卖方                                      │
├─────────────────────────────────────────────────────────────────┤
│  2. 报价                                                         │
│     - 查看询价详情                                                │
│     - 填写价格、品牌、货期                                        │
│     - 询价单状态变为 quoted                                       │
│     （可多个供应商分别报价）                                       │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 报价回复
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│                         买方                                      │
├─────────────────────────────────────────────────────────────────┤
│  3. 比价分析                                                     │
│     - 按价格/货期/品牌排序                                        │
│     - 综合评估最优供应商                                          │
│                                                                   │
│  4. 确认下单                                                     │
│     - 选择某条报价确认                                            │
│     - 生成订单，状态变为 ordered                                  │
│                                                                   │
│  5. 关闭询价（可选）                                              │
│     - 不需要的询价可手动关闭                                      │
└─────────────────────────────────────────────────────────────────┘
```

## 状态流转

```
询价单状态: pending → quoted → ordered → closed
                                  ↗
               可从任何状态 → closed
```

## 典型场景

### 场景一：汽配采购

修理厂接到维修单，需要采购刹车片：
1. `inquiry create --product "前刹车片" --oe "04465-33471" --vehicle "丰田凯美瑞 2020" --quantity 4`
2. 联系多家供应商报价
3. `compare` 选出性价比最优
4. `order confirm` 下单

### 场景二：IT 设备采购

公司需要采购服务器：
1. `inquiry create --product "Dell R750xs" --quantity 10 --note "需要3年保修"`
2. 收集各渠道报价
3. 比价后确认

### 场景三：工业品询价

工厂需要采购轴承：
1. `inquiry create --product "深沟球轴承" --oe "SKF-6205-2RS" --quantity 200`
2. 多家轴承经销商报价
3. 综合价格和货期决策
